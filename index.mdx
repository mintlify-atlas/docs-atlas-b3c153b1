GoodJob is a multithreaded, Postgres-based Active Job backend for Ruby on Rails. Designed for maximum compatibility with Rails, Active Job, and Postgres, it delivers reliable job execution with near-zero configuration — no Redis required.

## Why GoodJob?

GoodJob is designed to be the best Active Job backend for teams who already run Postgres and want simplicity without sacrificing power.

<CardGroup cols={2}>
  <Card title="Built for Rails" icon="gem">
    Fully adopts Rails threading and code execution guidelines. Works with `schema.rb` — no `structure.sql` required.
  </Card>
  <Card title="Backed by Postgres" icon="database">
    Uses Postgres ACID transactions, session-level advisory locks for run-once safety, and LISTEN/NOTIFY for low latency.
  </Card>
  <Card title="Fully Featured" icon="star">
    Cron-style scheduled jobs, batches, concurrency and throttling controls, bulk enqueue, and job pausing.
  </Card>
  <Card title="Flexible Deployment" icon="server">
    Run within your web process for economy or scale as an independent CLI process across environments.
  </Card>
</CardGroup>

## Get started in minutes

<Steps>
  <Step title="Add the gem">
    ```bash
    bundle add good_job
    ```
  </Step>
  <Step title="Run the install generator">
    ```bash
    bin/rails g good_job:install
    bin/rails db:migrate
    ```
  </Step>
  <Step title="Configure the queue adapter">
    ```ruby config/application.rb
    config.active_job.queue_adapter = :good_job
    ```
  </Step>
  <Step title="Enqueue your first job">
    ```ruby
    YourJob.perform_later
    ```
    In production, start the worker process:
    ```bash
    bundle exec good_job start
    ```
  </Step>
</Steps>
